The Internet

What is it?

The internet is just a bunch of connected computers.

But what about "the cloud"?

The cloud is just a bunch of connected computers that you're allowed to store your data on.

Clients and Servers

When a user types an address into a browser, their computer makes a request to another computer that is running software that can retrieve and send back (serve) files.

Github

What is Github?

“Git (/ɡɪt/) is a distributed revision control system with an emphasis on speed, data integrity, and support for distributed, non-linear workflows.”
  • Saves all file revisions for easy rollbacks
  • A central place for code collaboration
  • Open Source
  • A great way to host a simple website (Github Pages)

Git Workflow

Make a repository folder

If you don't have one already, create a folder on your computer where you keep all of your repositories (even though we only have 1 now).

mkdir ~/Sites

Now move into that directory:

cd ~/Sites

Git Workflow

Downloading your repository

Finally, download the entire contents of your repository using Git's "clone" comman: 

git clone git@github.com:username/username.github.io.git

Change something in your index.html!

Git Workflow

Uploading to your repository

Uploading files to your repository is called "pushing", and a push can contain several groups of files called "commits". Commits have comments attached to them that tells the world what was changed in those files.

Git Workflow

Push example

  • Commit: "created a new logo"
    • index.html
    • logo.png

  • Commit: "changed background color and title"
    • styles.css
    • index.html

Git Workflow

Adding/Staging Files

Files can be added to a commit one-by-one: 

git add index.html

Or by using wildcards.
To add every changed file, use the following: 

git add *

Git Workflow

Commiting Files

Once you've added all the files you want for a commit, create the commit using the following: 

git commit -m "my awesome commit message"

It is very difficult to "uncommit" it, so make sure everything is right!

Git Workflow

Pushing Files

When you have one or many commits ready for upload, simply type: 

git push

Git Workflow

Bonus: Pulling Files

If the remote repository has changes that you don't have locally (like if someone else pushed changes or you edited something using the web interface), it's very easy to update your local copy: 

git pull

Running the Site Locally

Running the Site Locally

So you've got updating files under your belt. But what if you want to view the changes in a browser BEFORE you push to the live site? These next few steps will get your computer set up as a local server to view these files.

Installing the Gem

Github Pages are served using Ruby. A Gemfile contains all of the Ruby dependencies needed in order to run the site. This only needs to be done once to initialize a site. 

gem install github-pages

Actually Running the Site

Now, all you need to do any time you want to serve the site is type: 

bundle exec jekyll serve --watch

Look in your browser!

You should be able to see your site at the following URL: 

http://localhost:4000/
